01 — About

What is PassportPath?

PassportPath is a single-file, client-side web application that provides an overview of visa requirements and entry conditions for approximately 50 nationalities across 60 destination countries. It is designed as an educational and orientation tool — a starting point for travel research, not a definitive legal resource.

The application runs entirely in the browser with no backend, no API calls, and no live data feed. All data is hardcoded at the time of build and reflects conditions as understood during development (2024–2025).

03 — Data Accuracy & Limitations

Known Limitations of the Data

Visa and entry requirements are among the most frequently changing categories of international law. The following limitations apply to all data in this application:

  • 📅
    Static, Time-Stamped DataThe data is hardcoded and was accurate (to the best of the author's knowledge) around 2024–2025. It is not updated automatically or in real time.
  • 🔁
    Reciprocity & Bilateral ChangesCountries can and do revise visa-free agreements, introduce new electronic authorisation requirements (such as ETAs or eTAs), or suspend access with little notice — often in response to diplomatic events.
  • 🆔
    Passport Variant DifferencesSome countries issue passports with different privileges (e.g. diplomatic vs. ordinary). Dual nationals may face additional rules. This application does not account for these distinctions.
  • 🚧
    Entry Conditions Beyond VisasEven where visa-free access applies, countries may require proof of onward travel, sufficient funds, vaccinations, travel insurance, valid return tickets, or may deny entry at the border officer's discretion.
  • 🌐
    Territorial Disputes & Special RegionsSome territories (e.g. Taiwan, Kosovo, Palestine) have complex status. This application uses simplified country codes and may not reflect the full legal or political complexity.
  • ✈
    Transit vs. EntryTransit visa requirements (when transiting through a country's airport without clearing immigration) are not covered and are handled differently from entry visas.

  • 🌐

ℹ Pro Tip

06 — Deployment

How to Deploy

PassportPath is a fully self-contained single HTML file. No build tools, frameworks, or servers are required.

✓ GitHub Pages (Recommended)
1. Fork or clone this repository.
2. Go to Settings → Pages in your GitHub repository.
3. Set the source branch to main and the folder to / (root).
4. GitHub will publish the app at https://[your-username].github.io/[repo-name]/passportpath.html
# Clone the repo git clone https://github.com/[your-username]/passportpath.git cd passportpath # No build step needed — open directly in browser open passportpath.html # Or serve locally with any static server npx serve . python3 -m http.server 8080
